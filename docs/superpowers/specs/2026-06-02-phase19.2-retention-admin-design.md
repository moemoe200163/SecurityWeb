# Phase 19.2 — Retention Admin (Status, Run, Dry-run)

## Overview

Expose retention operations to admins through a dedicated endpoint set: a status query that surfaces row counts, last-run timestamp, last result, and the active policy; a run endpoint that executes the existing `runRetentionCleanup` utility; and a dry-run mode that previews impact without mutating data. Replace the legacy `POST /api/admin/retention/cleanup` (which lives in `admin.ts` and uses the vague `action: 'cleanup'` audit code) with new endpoints under a clean namespace, while preserving the legacy route for backward compatibility.

**Out of scope**: scheduled/cron-style automatic execution (deferred to 19.4+); per-table retention policy overrides via UI; data export before deletion.

## Goals

1. Admins can query retention status: row counts per governed table, last-run timestamp, last result counts, and active policy values.
2. Admins can trigger a real retention run with the same defaults as the existing utility.
3. Admins can preview impact via `?dryRun=true` (no mutations).
4. New audit log entries use the standardized `action: 'retention_run'` code.
5. Legacy `POST /api/admin/retention/cleanup` continues to work unchanged (its `action: 'cleanup'` audit code is preserved for backward compatibility; new endpoints are recommended).

## Non-goals

- Automatic scheduled execution (cron).
- Per-tenant retention policy overrides.
- Pre-deletion data export.
- Retention policy changes via UI (policy values stay hardcoded in `utils/retention.ts` defaults).

## API Endpoints

All endpoints require `apiKeyAuth` + `requireAdmin` middleware.

| Method | Path | Purpose | Audit action |
|--------|------|---------|--------------|
| `GET`  | `/api/admin/retention/status` | Snapshot of current state | (none — read) |
| `POST` | `/api/admin/retention/run` | Execute cleanup (supports `?dryRun=true`) | `retention_run` |
| `POST` | `/api/admin/retention/cleanup` | **Legacy** (preserved) | `cleanup` (unchanged) |

### Status response

```typescript
{
  counts: {
    auditLog: number;       // prisma.auditLog.count()
    toolExecution: number;  // prisma.toolExecution.count()
    bgpUpdate: number;      // prisma.bgpUpdate.count()
  },
  lastRunAt: string | null;        // ISO timestamp from most recent audit_log action='retention_run' or 'cleanup'
  lastResult: {
    auditLogsDeleted: number;
    toolExecutionsTrimmed: number;
    bgpUpdatesDeleted: number;
  } | null,                        // parsed from audit_log details of the most recent run
  policy: {
    auditLogDays: 90;
    toolExecutionDays: 30;
    bgpUpdateDays: 7;
  }                                 // hardcoded defaults from utils/retention.ts
}
```

`lastRunAt` / `lastResult` query: `prisma.auditLog.findFirst({ where: { action: { in: ['retention_run', 'cleanup'] } }, orderBy: { createdAt: 'desc' } })`. If `details` is JSON, parse and return counts; if not, return `lastResult: null`.

### Run request

```
POST /api/admin/retention/run            → execute for real
POST /api/admin/retention/run?dryRun=true → preview only
```

Optional body (overrides policy for this run):

```typescript
{
  auditLogDays?: number;       // default 90
  toolExecutionDays?: number;  // default 30
  bgpUpdateDays?: number;      // default 7
}
```

### Run response (execute)

```typescript
{
  mode: 'execute',
  result: {
    auditLogsDeleted: number;
    toolExecutionsTrimmed: number;
    bgpUpdatesDeleted: number;
  },
  ranAt: string  // ISO
}
```

### Run response (dry-run)

```typescript
{
  mode: 'dry-run',
  preview: {
    auditLogsWouldDelete: number;
    toolExecutionsWouldTrim: number;
    bgpUpdatesWouldDelete: number;
  },
  ranAt: string
}
```

## Utility Changes — `backend/src/utils/retention.ts`

Add a `mode` parameter to `runRetentionCleanup` and supporting functions. Default is `'execute'`; `'preview'` uses `count` instead of `deleteMany` / `updateMany`.

### Updated signatures

```typescript
export type RetentionMode = 'execute' | 'preview';

export interface RetentionConfig {
  auditLogDays?: number;
  toolExecutionDays?: number;
  bgpUpdateDays?: number;
  mode?: RetentionMode;  // default 'execute'
}

export interface RetentionResult {
  auditLogsDeleted: number;
  toolExecutionsTrimmed: number;
  bgpUpdatesDeleted: number;
}

export interface RetentionPreview {
  auditLogsWouldDelete: number;
  toolExecutionsWouldTrim: number;
  bgpUpdatesWouldDelete: number;
}

// Returns either RetentionResult (execute) or RetentionPreview (preview)
export async function runRetentionCleanup(
  config?: RetentionConfig
): Promise<RetentionResult | RetentionPreview>
```

### Internal implementation pattern

```typescript
async function cleanupAuditLogs(days = 90, mode: RetentionMode = 'execute'): Promise<number> {
  const cutoff = computeCutoff(days);
  if (mode === 'preview') {
    return prisma.auditLog.count({ where: { createdAt: { lt: cutoff } } });
  }
  const result = await prisma.auditLog.deleteMany({ where: { createdAt: { lt: cutoff } } });
  return result.count;
}
// Similar pattern for trimToolExecutionOutput and cleanupBgpUpdates
```

`runRetentionCleanup` aggregates results and returns `RetentionResult` (when `mode === 'execute'`) or `RetentionPreview` (when `mode === 'preview'`). Callers can discriminate by shape or by checking `mode` in the response wrapper (preferred — the route handler builds the wrapper).

**Behavior preservation**: with `mode` defaulting to `'execute'` and the existing call site in `admin.ts` not passing `mode`, the legacy endpoint's behavior is unchanged.

## Backend Implementation

### `backend/src/routes/adminRetention.ts` (new)

```typescript
import type { FastifyInstance } from 'fastify';
import { z } from 'zod';
import { apiKeyAuth } from '../middleware/apiKeyAuth.js';
import { requireAdmin } from '../middleware/rbac.js';
import { prisma } from '../db/client.js';
import { runRetentionCleanup, type RetentionMode } from '../utils/retention.js';
import { sanitizeAuditDetails } from '../utils/sanitize.js';

const POLICY_DEFAULTS = { auditLogDays: 90, toolExecutionDays: 30, bgpUpdateDays: 7 };

const runBodySchema = z.object({
  auditLogDays: z.number().int().positive().optional(),
  toolExecutionDays: z.number().int().positive().optional(),
  bgpUpdateDays: z.number().int().positive().optional(),
});

export async function adminRetentionRoutes(fastify: FastifyInstance): Promise<void> {
  // GET /admin/retention/status
  fastify.get(
    '/retention/status',
    { preHandler: [apiKeyAuth, requireAdmin] },
    async (_request, reply) => {
      try {
        const [auditLog, toolExecution, bgpUpdate, lastEntry] = await Promise.all([
          prisma.auditLog.count(),
          prisma.toolExecution.count(),
          prisma.bgpUpdate.count(),
          prisma.auditLog.findFirst({
            where: { action: { in: ['retention_run', 'cleanup'] } },
            orderBy: { createdAt: 'desc' },
          }),
        ]);

        let lastResult: RetentionResult | null = null;
        if (lastEntry?.details) {
          const d = lastEntry.details as Record<string, unknown>;
          // Handle two shapes:
          //   Legacy (action='cleanup'):    details = { auditLogsDeleted, toolExecutionsTrimmed, bgpUpdatesDeleted }
          //   New     (action='retention_run'): details = { mode, result: { auditLogsDeleted, ... }, config }
          const source = (d.result as Record<string, unknown> | undefined) ?? d;
          if (typeof source.auditLogsDeleted === 'number') {
            lastResult = {
              auditLogsDeleted: source.auditLogsDeleted,
              toolExecutionsTrimmed: source.toolExecutionsTrimmed ?? 0,
              bgpUpdatesDeleted: source.bgpUpdatesDeleted ?? 0,
            };
          }
        }

        return reply.send({
          counts: { auditLog, toolExecution, bgpUpdate },
          lastRunAt: lastEntry?.createdAt.toISOString() ?? null,
          lastResult,
          policy: POLICY_DEFAULTS,
        });
      } catch (error) {
        fastify.log.error(error);
        return reply.status(500).send({ error: 'Failed to fetch retention status' });
      }
    }
  );

  // POST /admin/retention/run
  fastify.post(
    '/retention/run',
    { preHandler: [apiKeyAuth, requireAdmin] },
    async (request, reply) => {
      try {
        const dryRun = (request.query as { dryRun?: string }).dryRun === 'true';
        const body = runBodySchema.parse(request.body ?? {});
        const mode: RetentionMode = dryRun ? 'preview' : 'execute';

        const out = await runRetentionCleanup({
          auditLogDays: body.auditLogDays,
          toolExecutionDays: body.toolExecutionDays,
          bgpUpdateDays: body.bgpUpdateDays,
          mode,
        });

        const ranAt = new Date().toISOString();

        // Always audit (both dry-run and real)
        await prisma.auditLog.create({
          data: {
            userId: request.user!.id,
            action: 'retention_run',
            resourceType: 'retention',
            details: sanitizeAuditDetails({
              mode,
              ...(dryRun
                ? { preview: out }
                : { result: out }),
              config: body,
            }),
          },
        });

        if (dryRun) {
          return reply.send({ mode: 'dry-run', preview: out, ranAt });
        }
        return reply.send({ mode: 'execute', result: out, ranAt });
      } catch (error) {
        fastify.log.error(error);
        return reply.status(500).send({ error: 'Failed to run retention' });
      }
    }
  );
}
```

## Error Handling

| Scenario | HTTP | code (audit) | error | Side effect |
|----------|------|-------------|-------|-------------|
| Non-admin | 403 | `forbidden` | "Admin access required" | audit log |
| Invalid body (negative days) | 400 | — | "Invalid retention parameters" | none |
| DB error on count | 500 | `internal_error` | "Failed to fetch retention status" | none |
| DB error on run | 500 | `internal_error` | "Failed to run retention" | audit log: `retention_run_failed` |
| Partial failure (one table fails mid-run) | 500 | `partial_failure` | "Retention partially failed" | audit log: `retention_run_partial` with details of which succeeded |
| Dry-run requested but no data to delete | 200 | — | (success, zero counts) | audit log written |

**Partial failure handling**: switch `runRetentionCleanup` from `Promise.all` to `Promise.allSettled`. If any rejection, the route handler returns 500 with `{ errors: [{ table: 'auditLog' \| 'toolExecution' \| 'bgpUpdate', message }], partial: { auditLogsDeleted, toolExecutionsTrimmed, bgpUpdatesDeleted } }` (where partial counts are 0 for any table that failed). The audit log entry uses `action: 'retention_run_partial'` with sanitized error details.

Trade-off vs sequential: `allSettled` keeps parallel execution (faster) and gives full per-table status. Sequential would be slightly slower and is not needed.

Client-side: 403 → toast "Permission denied". 400 → inline form error. 500 → toast with retry button.
